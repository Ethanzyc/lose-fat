## 流程

### 第一步：用户识别

1. 使用 Bash 工具扫描用户档案目录：`ls /Users/zhuyuchen/ai/lose-fat/profiles/*.json 2>/dev/null`
2. 根据结果进入对应分支：

**无档案文件时：**
- 告知用户「未找到已有档案，将为您创建新档案」
- 进入「新建档案」流程

**有档案文件时：**
- 列出所有已有用户（读取每个 JSON 的 name 字段）
- 同时提供「新建用户」选项
- 使用 AskUserQuestion 工具让用户选择
- 选择已有用户 → 读取该用户档案，进入「回访判断」
- 选择新建用户 → 进入「新建档案」流程

### 第二步：新建档案

使用 AskUserQuestion 工具依次收集以下信息（每次 1-2 个问题）：

**基本信息：**
- 姓名
- 年龄（25-50 岁，超出范围时给出提示）
- 性别
- 身高（cm）
- 体重（kg）

**健康数据（一次展示所有字段，未知留空）：**

使用 AskUserQuestion 工具，提示用户：「请提供以下检查指标，如果没有检查过的项目可以不填。」

| 字段 | 说明 | 单位 |
|------|------|------|
| 甘油三酯（TG） | 必填 | mmol/L |
| 总胆固醇（TC） | 可选 | mmol/L |
| 高密度脂蛋白（HDL） | 可选 | mmol/L |
| 低密度脂蛋白（LDL） | 可选 | mmol/L |
| 丙氨酸转氨酶（ALT） | 可选 | U/L |
| 天冬氨酸转氨酶（AST） | 可选 | U/L |
| 空腹血糖 | 可选 | mmol/L |
| 肝脏 B 超结果 | 可选 | 正常/轻度/中度/重度 |

收集完成后：
1. 计算 BMI = 体重(kg) / (身高(m))²
2. 将所有数据保存为 JSON 文件到 `/Users/zhuyuchen/ai/lose-fat/profiles/{name_pinyin}_{date}.json`
3. 进入「健康评估」

### 第四步：健康评估

读取 `/Users/zhuyuchen/ai/lose-fat/skills/lose-fat/knowledge/triglycerides.md` 和 `/Users/zhuyuchen/ai/lose-fat/skills/lose-fat/knowledge/fatty-liver.md`。

基于用户数据输出：

**4.1 指标解读：**

对每项有值的指标进行解读，格式如下：

```
## 指标解读

| 指标 | 您的数值 | 参考范围 | 状态 | 分级 |
|------|---------|---------|------|------|
| 甘油三酯 | 3.2 mmol/L | < 1.7 mmol/L | 偏高 | 升高 |
| BMI | 27.8 | 18.5-23.9 | 偏高 | 超重 |
```

BMI 分级标准（中国）：
- 偏瘦：< 18.5
- 正常：18.5-23.9
- 超重：24.0-27.9
- 肥胖：≥ 28.0

**4.2 风险评估：**

综合所有指标判断代谢综合征风险等级：

- **低风险**：仅甘油三酯轻度升高，无其他异常
- **中风险**：甘油三酯升高 + 超重/肥胖，或合并 1-2 项其他指标异常
- **高风险**：甘油三酯显著升高 + 多项指标异常 + 脂肪肝中重度

**4.3 核心建议：**

列出 3-5 条最关键的改善建议，按优先级排列。建议必须具体、可操作。

**4.4 免责声明**

评估完成后进入「12 周计划生成」。

### 第五步：12 周减脂计划生成

读取 `/Users/zhuyuchen/ai/lose-fat/skills/lose-fat/knowledge/nutrition-guide.md`、`/Users/zhuyuchen/ai/lose-fat/skills/lose-fat/knowledge/exercise-guide.md` 和 `/Users/zhuyuchen/ai/lose-fat/skills/lose-fat/knowledge/lifestyle.md`。

首先计算用户的：
- BMR（使用 Mifflin-St Jeor 公式）
- TDEE（假设初始活动系数 1.2，久坐）
- 每日目标热量（TDEE - 500 kcal，不低于最低限制）

然后按 3 个阶段生成计划：

#### 阶段 1：适应期（第 1-4 周）

**目标：** 调整饮食结构，建立运动习惯，体重下降 2-4%

**饮食方案：**
- 每日热量目标（具体数值）
- 三大营养素比例和克数
- 1 周示例食谱（早/午/晚/加餐，含具体食物和份量）
- 食物替换建议

**运动方案：**
- 以低-中等强度有氧运动为主
- 每周运动安排表（日期 + 项目 + 时长）
- 新手从快走开始，逐步过渡到慢跑

**生活习惯：**
- 2-3 条最关键的习惯调整建议

**阶段检查点：**
- 第 2 周末：体重下降 1-2%，运动习惯初步建立
- 第 4 周末：体重累计下降 2-4%，能够完成 30 分钟中等强度有氧

#### 阶段 2：强化期（第 5-8 周）

**目标：** 提升运动强度，精细化营养管理，体重累计下降 5-8%

**饮食方案：**
- 根据阶段 1 进展微调热量目标
- 增加蛋白质比例至 30%
- 更换食谱（新的一周示例）

**运动方案：**
- 引入抗阻训练（每周 2 次）
- 有氧运动强度提升至中等
- 每周运动安排表

**阶段检查点：**
- 第 6 周末：体重累计下降 4-6%，抗阻训练动作掌握
- 第 8 周末：体重累计下降 5-8%，肝功能指标可能开始改善

#### 阶段 3：巩固期（第 9-12 周）

**目标：** 巩固习惯，防止反弹，甘油三酯目标降至正常范围

**饮食方案：**
- 热量缺口缩小至 200-300 kcal（接近维持期）
- 教授长期可持续的饮食模式
- 更换食谱（新的一周示例）

**运动方案：**
- 有氧 + 抗阻均衡安排
- 每周运动安排表
- 引入灵活性训练（拉伸、瑜伽）

**阶段检查点：**
- 第 10 周末：体重累计下降 8-10%，各项指标明显改善
- 第 12 周末：完成计划，建议复查血脂和肝功能

**输出完整计划后进入「导出」步骤。**

### 第六步：导出

2. 询问用户：「是否需要将此方案导出为文件？导出后将保存为 markdown 文件。」
3. 如果用户选择导出：
   - 将完整的评估和计划内容写入 `/Users/zhuyuchen/ai/lose-fat/output/{用户名}_{日期}_减脂方案.md`
   - 告知用户文件路径

### 第七步：进度回访

用户选择「报告进展」时执行此步骤。

1. 展示用户当前档案中的基本信息和上次记录
2. 使用 AskUserQuestion 工具收集新数据：

| 字段 | 说明 |
|------|------|
| 当前体重 | kg |
| 本阶段运动完成情况 | 大致百分比 |
| 饮食执行情况 | 良好/一般/较差 |
| 主观感受 | 自由文本 |
| 新的检查指标（如有） | TG、ALT 等 |

3. **生成进度报告：**
   - 体重变化趋势（当前 vs 起始 vs 上次）
   - 目标达成率（对比阶段检查点）
   - 表扬亮点 + 需要改进的方面

4. **调整计划：**
   - 进展超前 → 可适当提升下一阶段难度
   - 进展正常 → 按原计划继续
   - 进展落后 → 分析原因，调整方案（可能是热量目标需要重新计算，或运动方案需要简化）

5. 更新用户档案 JSON 文件（添加 progress_records 条目，更新 plan.current_phase）

6. 输出免责声明
